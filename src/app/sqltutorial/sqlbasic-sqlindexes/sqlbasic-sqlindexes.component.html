SQL Indexes

What is an Index in SQL?

An Index in SQL is a database object that improves the speed of data retrieval operations. It acts like an index in a book, allowing faster searches without scanning the entire table.

Types of Indexes

1. Clustered Index

A clustered index determines the physical order of data in a table. Each table can have only one clustered index. 

CREATE CLUSTERED INDEX IX_Employee_ID ON Employees(EmployeeID);

2. Non-Clustered Index

A non-clustered index creates a separate structure from the table, storing pointers to data. A table can have multiple non-clustered indexes. 

CREATE NONCLUSTERED INDEX IX_Employee_Name ON Employees(LastName);

3. Unique Index

Ensures that indexed column(s) contain distinct values, preventing duplicates. 

CREATE UNIQUE INDEX IX_Unique_Email ON Employees(Email);

4. Full-Text Index

Used for searching large text fields efficiently.

CREATE FULLTEXT INDEX ON Articles(Title, Content) KEY INDEX PK_ArticleID;

5. Composite Index

Created on multiple columns to optimize multi-condition queries. 

CREATE INDEX IX_Employee_Department ON Employees(DepartmentID, LastName);

6. Filtered Index

Created for a specific subset of rows, reducing storage overhead. 

CREATE INDEX IX_Active_Employees ON Employees(EmployeeID) WHERE Status = 'Active';

7. Spatial Index

Used for efficient retrieval of spatial (geographical) data. 

CREATE SPATIAL INDEX IX_Location ON Properties(Location);
