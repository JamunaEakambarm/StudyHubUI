SQL Joins

An SQL JOIN is a powerful feature in relational databases that allows you to combine rows from two or more tables based on a related column between them. Joins are essential when working with normalized databases, where data is distributed across multiple tables. For example, in a banking system, customers and transactions might be stored in separate tables, but using a JOIN, you can retrieve a combined view showing which customer made which transaction. SQL provides different types of joins to handle various data relationships, such as:

  • INNER JOIN
  • LEFT JOIN (or LEFT OUTER JOIN)
  • RIGHT JOIN (or RIGHT OUTER JOIN)
  • FULL JOIN (or FULL OUTER JOIN)
  • CROSS JOIN
  • SELF JOIN

INNER JOIN

An INNER JOIN retrieves only the rows where there is a match in both tables.

Use Case: If you want to find the details of transactions (from a transaction table) for

specific bank accounts (from an accounts table), you would use an INNER JOIN. Only the accounts that have

corresponding transactions will be shown.

SELECT customers.customer_id, customers.customer_name, transactions.transaction_id, transactions.amount FROM customers INNER JOIN transactions ON customers.customer_id = transactions.customer_id;

LEFT JOIN (or LEFT OUTER JOIN)

A LEFT JOIN returns all rows from the left table and the

matched rows from the right table. If there is no match, the result is NULL for columns from the right table.

Use Case: You might want to show a list of all customers and their loans. Some customers may

have loans, but you still want to display them, with NULL for loan details where there is no loan.

SELECT customers.customer_id, customers.customer_name, loans.loan_id, loans.loan_amount FROM customers LEFT JOIN loans ON customers.customer_id = loans.customer_id;

RIGHT JOIN (or RIGHT OUTER JOIN)

A RIGHT JOIN is the opposite of a LEFT JOIN. It returns all rows from the right table and the matched rows from the left table. If there is no match, the result is NULL for columns from the left table.

Use Case: Imagine you have a list of loans and you want to check for customers that have no related loan records. You can use a RIGHT JOIN if you're mainly concerned with the loan data.


        SELECT loans.loan_id, loans.loan_amount, customers.customer_id, customers.customer_name
        FROM loans
        RIGHT JOIN customers ON loans.customer_id = customers.customer_id;

FULL JOIN (or FULL OUTER JOIN)

A FULL JOIN returns all rows when there is a match in either the left or right table. If there is no match, NULL is returned for non-matching rows from both tables.

Use Case: Suppose you want to retrieve all accounts and all transactions, but some accounts may not have transactions, or some transactions might not be associated with an account. You would use a FULL JOIN.

SELECT accounts.account_id, accounts.balance, transactions.transaction_id, transactions.amount FROM accounts FULL JOIN transactions ON accounts.account_id = transactions.account_id;

CROSS JOIN

A CROSS JOIN returns the Cartesian product of the two tables. This means every row in the left table is combined with every row in the right table. Be cautious with this join as it can produce very large datasets.

Use Case: While not frequently used, a CROSS JOIN could be used if you need to calculate all possible combinations of products and customers, such as generating all possible loan-product combinations.

SELECT customers.customer_id, loans.loan_id FROM customers CROSS JOIN loans;

SELF JOIN

A SELF JOIN is when a table is joined with itself. This is useful when you need to compare rows within the same table.

Use Case: For example, if you wanted to compare transactions for the same customer over time, you could use a self join on the transactions table.

SELECT t1.transaction_id, t1.amount, t2.transaction_id, t2.amount FROM transactions t1 JOIN transactions t2 ON t1.customer_id = t2.customer_id WHERE t1.transaction_date < t2.transaction_date;

Summary of Join Types:

  • INNER JOIN ‚Äì Returns rows with matching values in both tables.
  • LEFT JOIN ‚Äì Returns all rows from the left table and matched rows from the right table.
  • RIGHT JOIN ‚Äì Returns all rows from the right table and matched rows from the left table.
  • FULL JOIN ‚Äì Returns all rows when there is a match in either left or right table.
  • CROSS JOIN ‚Äì Returns the Cartesian product of both tables (every combination of rows).
  • SELF JOIN ‚Äì Joins a table with itself to compare rows within the same table.
