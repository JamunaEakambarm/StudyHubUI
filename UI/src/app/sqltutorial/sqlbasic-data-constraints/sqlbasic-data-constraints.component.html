Data Constraints in SQL

Data constraints are rules applied to table columns to ensure data accuracy and integrity.

Types of Constraints:

  • PRIMARY KEY - Uniquely identifies each record in a table.
  • FOREIGN KEY - Ensures a relationship between two tables.
  • UNIQUE - Ensures all values in a column are distinct.
  • NOT NULL - Prevents null (empty) values in a column.
  • CHECK - Restricts values based on a condition.
  • DEFAULT - Assigns a default value if none is provided.

PRIMARY KEY Constraint

Ensures that each record is unique and not null.

Example (Bank Customers Table with Primary Key):

CREATE TABLE Customers ( CustomerID INT PRIMARY KEY, FirstName VARCHAR(50), LastName VARCHAR(50), Email VARCHAR(100) UNIQUE);

FOREIGN KEY Constraint

Links one table to another, ensuring referential integrity.

Example (Accounts Table Linked to Customers Table):

CREATE TABLE Accounts ( AccountID INT PRIMARY KEY, CustomerID INT, AccountType VARCHAR(20), Balance DECIMAL(10,2), FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID));

UNIQUE Constraint

Prevents duplicate values in a column.

Example (Ensuring Unique Email IDs for Customers):

ALTER TABLE Customers ADD CONSTRAINT UniqueEmail UNIQUE (Email);

NOT NULL Constraint

Prevents null (empty) values in a column.

Example (Ensuring Customers Must Have an Email):

ALTER TABLE Customers ALTER COLUMN Email SET NOT NULL;

CHECK Constraint

Restricts column values based on a condition.

Example (Ensuring Minimum Balance in a Bank Account):

ALTER TABLE Accounts ADD CONSTRAINT CheckBalance CHECK (Balance >= 1000);

DEFAULT Constraint

Provides a default value if no value is provided.

Example (Setting Default Account Type as 'Savings'):

ALTER TABLE Accounts ADD COLUMN AccountType VARCHAR(20) DEFAULT 'Savings';
