SQL Stored Procedures

A Stored Procedure is a precompiled set of SQL statements that can be executed as a single unit. It allows you to store and reuse SQL code, making database operations more efficient and secure. Stored procedures can accept input parameters, return output values, and handle business logic inside the database.

Advantages of Stored Procedures:

  • Improves Performance
  • Reusability
  • Security
  • Reduces Network Traffic
  • Encapsulates Business Logic

Basic Syntax:

CREATE PROCEDURE ProcedureName AS BEGIN -- SQL Statements END;

Types of Stored Procedures:

1. Stored Procedure Without Parameters

Example: Retrieve all employees

CREATE PROCEDURE GetAllEmployees AS BEGIN SELECT * FROM Employees; END;

2. Stored Procedure With Input Parameters

Example: Get employee details by department

CREATE PROCEDURE GetEmployeesByDepartment @DepartmentID INT AS BEGIN SELECT * FROM Employees WHERE DepartmentID = @DepartmentID; END;

3. Stored Procedure With Output Parameter

Example: Get total employees count

CREATE PROCEDURE GetTotalEmployees @TotalCount INT OUTPUT AS BEGIN SELECT @TotalCount = COUNT(*) FROM Employees; END;

4. Stored Procedure With Multiple Parameters

Example: Retrieve employees based on department and job title

CREATE PROCEDURE GetEmployeesByDeptAndJob @DepartmentID INT, @JobTitle VARCHAR(50) AS BEGIN SELECT * FROM Employees WHERE DepartmentID = @DepartmentID AND JobTitle = @JobTitle; END;

5. Stored Procedure With Transactions

Example: Transferring funds between accounts

CREATE PROCEDURE TransferFunds @SenderID INT, @ReceiverID INT, @Amount DECIMAL(10,2) AS BEGIN BEGIN TRANSACTION; UPDATE Accounts SET Balance = Balance - @Amount WHERE AccountID = @SenderID; UPDATE Accounts SET Balance = Balance + @Amount WHERE AccountID = @ReceiverID; COMMIT TRANSACTION; END;

Use Cases of Stored Procedures:

  • Data Validation and Business Rules
  • Batch Processing
  • Data Security
  • Logging and Auditing
  • Complex Data Manipulation
