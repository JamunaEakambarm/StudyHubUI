SQL Views

What is a View?

A View in SQL is a virtual table based on a query result. It simplifies complex queries and enhances data security.

Types of Views

  • Simple View ‚Äì Based on a single table.
  • Complex View ‚Äì Involves multiple tables and aggregations.
  • Materialized View ‚Äì Physically stores query results.
  • Inline View ‚Äì A subquery within the `FROM` clause.

Examples of Views

Simple View

Shows only active customers from the Customer table.

CREATE VIEW ActiveCustomers AS SELECT CustomerID, FirstName, LastName, AccountStatus FROM Customers WHERE AccountStatus = 'Active';

Complex View

Displays customer names and their total account balance.

CREATE VIEW CustomerBalance AS SELECT c.CustomerID, c.FirstName, c.LastName, SUM(a.Balance) AS TotalBalance FROM Customers c JOIN Accounts a ON c.CustomerID = a.CustomerID GROUP BY c.CustomerID, c.FirstName, c.LastName;

Materialized View

Stores high-value transactions for faster retrieval.

CREATE MATERIALIZED VIEW HighValueTransactions AS SELECT TransactionID, CustomerID, Amount, TransactionDate FROM Transactions WHERE Amount > 50000;

Inline View

Finds the highest transaction amount per customer.

SELECT CustomerID, FirstName, LastName, MaxTransaction FROM ( SELECT c.CustomerID, c.FirstName, c.LastName, MAX(t.Amount) AS MaxTransaction FROM Customers c JOIN Transactions t ON c.CustomerID = t.CustomerID GROUP BY c.CustomerID, c.FirstName, c.LastName ) AS TempView WHERE MaxTransaction > 50000;

Benefits of Views

  • Data Security ‚Äì Restricts access to sensitive data.
  • Simplifies Queries ‚Äì Makes complex queries easier.
  • Performance Boost ‚Äì Materialized views improve speed.
  • Encapsulation ‚Äì Hides database complexity.
